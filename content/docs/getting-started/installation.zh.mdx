## Docker 部署（推荐）

对于不同的服务器网络环境，推荐根据需要选择以下镜像源：

| 镜像源 | 镜像地址 | 说明 |
|------|---------|-----|
| Docker Hub | `openaigcs/aigcs:latest` | 官方默认镜像 |
| GitHub GHCR | `ghcr.io/openaigcs/aigcs:latest` | GitHub 容器镜像 |
| 阿里云 (ACR) | `registry.cn-shanghai.aliyuncs.com/openaigcs/aigcs:latest` | 华东 2（上海）节点 |
| CNB 镜像源 | `docker.cnb.cool/openaigcs/aigcs:latest` | 腾讯云旗下 CNB 镜像源 |

### 使用 Docker Run

你可以直接替换下面命令中的镜像名称来使用加速镜像。

```bash
docker run -d \
  --name aigcs \
  -p 41905:41905 \
  -e JWT_SECRET="<your-secret>" \
  -e ENCRYPTION_KEY="<your-encryption-key>" \
  -v ./data:/app/data \
  openaigcs/aigcs:latest
```

### 使用 Docker Compose

创建 `compose.yml`：

```yaml
services:
  aigcs:
    image: openaigcs/aigcs:latest
    container_name: aigcs
    ports:
      - "41905:41905"
    environment:
      - JWT_SECRET=<change-this>
      - ENCRYPTION_KEY=<change-this>
      - NODE_ENV=production
    volumes:
      - ./data:/app/data
    healthcheck:
      test: ["CMD", "wget", "--spider", "http://localhost:41905/api/health"]
      interval: 30s
      timeout: 10s
      retries: 3
```

```bash
docker compose up -d
```

### 环境变量

| 变量 | 默认值 | 必填 | 说明 |
|------|--------|------|------|
| `PORT` | `41905` | 否 | HTTP 服务端口（leetspeak: aigcs → 41905） |
| `DATABASE_URL` | `file:./data/aigcs.db` | 否 | SQLite 数据库文件路径 |
| `JWT_SECRET` | — | ✅ | JWT 签名密钥（生产环境务必修改） |
| `ENCRYPTION_KEY` | `JWT_SECRET` | 否 | AES-256-GCM 加密密钥（建议与 JWT_SECRET 隔离） |
| `RATE_LIMIT_MAX` | `100` | 否 | 每个窗口最大请求数 |
| `RATE_LIMIT_WINDOW` | `60` | 否 | 速率限制窗口（秒） |
| `SMTP_HOST` | — | 否 | SMTP 服务器地址 |
| `SMTP_PORT` | `587` | 否 | SMTP 端口 |
| `SMTP_USER` | — | 否 | SMTP 用户名 |
| `SMTP_PASS` | — | 否 | SMTP 密码 |
| `SMTP_FROM` | — | 否 | 发件人地址 |
| `CAPTCHA_PROVIDER` | — | 否 | 验证码类型：`turnstile` / `recaptcha` / `geetest` / `none` |
| `TURNSTILE_SITE_KEY` | — | 否 | Cloudflare Turnstile Site Key |
| `TURNSTILE_SECRET_KEY` | — | 否 | Cloudflare Turnstile Secret Key |
| `RECAPTCHA_SITE_KEY` | — | 否 | Google reCAPTCHA Site Key |
| `RECAPTCHA_SECRET_KEY` | — | 否 | Google reCAPTCHA Secret Key |
| `GEETEST_ID` | — | 否 | 极验 ID |
| `GEETEST_KEY` | — | 否 | 极验 Key |
| `ALLOWED_ORIGINS` | `[]` | 否 | 允许的 CORS 来源（JSON 数组字符串） |
| `REGISTRATION_OPEN` | `false` | 否 | 是否开放注册 |
| `GLOBAL_SYSTEM_PROMPT` | — | 否 | 全局 AI 系统提示词 |

## 手动部署

### 前提条件

- Node.js >= 24.0.0
- pnpm >= 9.0.0

### 步骤

```bash
# 克隆仓库
git clone https://github.com/openaigcs/aigcs.git
cd aigcs

# 安装依赖
pnpm install

# 构建所有包
pnpm build

# 创建包含配置的 .env.local
echo 'JWT_SECRET=your-random-secret' > .env.local

# 启动服务器
pnpm --filter @aigcs/server start
```

服务器默认在端口 `41905` 启动。

### 进程管理器（PM2）

```bash
npm install -g pm2
pm2 start node --name aigcs -- packages/server/dist/index.js
pm2 save
pm2 startup
```

### Systemd 服务

```ini
[Unit]
Description=AIGCS Comment System
After=network.target

[Service]
Type=simple
User=node
WorkingDirectory=/opt/aigcs
ExecStart=/usr/local/bin/node packages/server/dist/index.js
Restart=always
Environment=NODE_ENV=production

[Install]
WantedBy=multi-user.target
```

## 反向代理

### Nginx

```nginx
server {
    listen 443 ssl;
    server_name comments.example.com;

    ssl_certificate /path/to/cert.pem;
    ssl_certificate_key /path/to/key.pem;

    location / {
        proxy_pass http://127.0.0.1:41905;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

### Caddy

```caddy
comments.example.com {
    reverse_proxy 127.0.0.1:41905
}
```
