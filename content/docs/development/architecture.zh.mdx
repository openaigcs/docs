## 概述

AIGCS 当前版本 (V1) 被定义为一个高度内聚的标准 Node.js Monorepo。主要聚焦在基于传统服务器端环境和 SQLite 本地文件数据库的稳定性上。为了方便未来的开发，结构虽然预留了对边缘函数 (Edge Functions，如腾讯云 EdgeOne) 的部署打包兼容（位于 `packages/edgeone`），但彻底的**云原生（Serverless/分布式架构）改造**被推迟至下一个大阶段。我们目前的理念是“把现在作为稳定的 V1 第一版”。

AIGCS 是一个单体仓库，包含 `packages/` 下的五个核心包和 `packages/plugins/` 下的插件：

```
packages/
├── core/         共享类型、Drizzle ORM 模式、常量
├── server/       Hono HTTP API 服务器、中间件、通知服务
├── admin/        基于 TanStack Router + React Query 构建的现代 SPA 管理面板
├── widget/       用于显示评论的 Web 组件（无框架依赖的 Shadow DOM）
└── edgeone/      为未来云原生 Edge Functions 预留的打包构建脚本
packages/plugins/
├── native/       访客评论插件
└── mastodon/     联邦宇宙/ActivityPub 集成
```

## 数据流

```
用户浏览器                    AIGCS 服务器                    AI 提供商
     │                              │                              │
     │  ── Widget 加载 ────────►    │                              │
     │  ◄── 返回评论 JSON ────────  │                              │
     │                              │                              │
     │  ── 获取页面内容 ──────►     │                              │
     │                              │  ── 生成提示词 ──────────►   │
     │                              │  ◄── AI 响应 ──────────────  │
     │  ◄── AI 评论已存储 ────────  │                              │
```

## 请求生命周期

1. **Widget** 通过网站上的 `<script>` 加载
2. Widget 从 **服务器** API 获取评论
3. 服务器检查页面缓存：
   - **已缓存**：立即返回现有的 AI 评论
   - **未缓存**：获取页面内容 → 发送给 AI 提供商 → 生成评论 → 缓存
4. 评论显示在 Widget 中（CDN 缓存 + ETag）

## 插件系统

插件通过钩子系统接入服务器生命周期：

```
beforeGenerate → generate → afterGenerate → beforeRender → afterRender
```

每个钩子接收一个上下文对象（`PluginHookContext`），可访问请求数据、评论和服务器配置。
