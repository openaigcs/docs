## 前提条件

- **Node.js** >= 24.0.0
- **pnpm** >= 9.0.0
- **Docker**（可选，用于数据库或全栈测试）

## 设置

```bash
# 克隆仓库
git clone https://github.com/openaigcs/aigcs.git
cd aigcs

# 安装所有依赖
pnpm install

# 构建所有包
pnpm build
```

## 运行服务

在每个独立的终端中启动各服务：

```bash
# 服务器（API + 管理面板）
pnpm --filter @aigcs/server dev

# 管理面板（独立的开发服务器，支持 HMR）
pnpm --filter @aigcs/admin dev

# Widget（监听模式）
pnpm --filter @aigcs/widget dev
```

## 环境变量

在项目根目录创建 `.env.local`：

```bash
echo 'JWT_SECRET=dev-secret-change-in-production' > .env.local
echo 'ENCRYPTION_KEY=dev-encryption-key' >> .env.local
```

所有可用选项请参见[配置参考](../reference/config)。

## 常用命令

```bash
# 构建指定包
pnpm --filter @aigcs/server build

# 运行测试
pnpm --filter @aigcs/core test

# 检查所有包的代码风格
pnpm lint

# TypeScript 类型检查
pnpm typecheck
```

## pnpm 升级规范

为了保持本地开发和 Docker 容器内构建版本的一致性，AIGCS 使用 `corepack` 锁定 pnpm 版本。当需要升级 pnpm 包管理器自身时，请遵循以下标准流程：

1. **更新配置文件**：打开根目录的 `package.json`，修改 `"packageManager"` 字段为您想要升级的目标版本（例如 `"pnpm@11.15.1"`）。
2. **激活与安装**：在项目根目录下执行以下命令，让 Corepack 拉取并激活新版本，同时更新锁定关系：
   ```bash
   corepack use pnpm@11.15.1
   ```
3. **刷新依赖锁文件**：执行 `pnpm install` 刷新 `pnpm-lock.yaml` 文件：
   ```bash
   pnpm install
   ```
4. **编译测试**：运行 `pnpm build` 确认项目在新的包管理器下能够正常构建，并在测试通过后将 `package.json` 与 `pnpm-lock.yaml` 一起提交。
