AIGCS 支持先进的免密登录 (Passkey) 和 GitHub、Google 社交账号登录。

## 密钥 (Passkey / WebAuthn)

Passkey 允许您通过生物特征验证（如 Apple Touch ID / Face ID, Windows Hello, Android 指纹）或密码管理器（如 Bitwarden, 1Password）进行安全快速的免密登录。

### 前提条件

- Passkey 要求安全上下文环境。它们**仅在** `HTTPS` 或本地开发环境 `localhost` 下工作。
- 无需额外的服务器环境变量配置。

### 绑定与启用

1. 使用传统的邮箱/用户名和密码登录 AIGCS 后台。
2. 前往 **个人设置** (点击左下角头像 -> 个人设置)。
3. 在 **密钥 (Passkeys)** 卡片下，点击 **添加 Passkey**。
4. 按照浏览器弹窗提示，验证您的指纹、面容或输入设备 PIN 码（或者调用您的 Bitwarden 等密码管理器保存密钥）。
5. 验证成功后，密钥将呈现在已注册列表里。在登录页面上，点击 **使用 Passkey 免密登录** 即可一键登录。

## 第三方快捷登录 (OAuth 2.0)

AIGCS 支持使用 GitHub 和 Google 账号进行一键快捷登录。

### 环境变量配置

请在您服务器的 `.env` 配置文件或系统环境变量中，配置以下参数来启用对应的登录选项：

```bash
# GitHub OAuth 配置
GITHUB_CLIENT_ID=您的_github_client_id
GITHUB_CLIENT_SECRET=您的_github_client_secret

# Google OAuth 配置
GOOGLE_CLIENT_ID=您的_google_client_id
GOOGLE_CLIENT_SECRET=您的_google_client_secret

# 可选: 强制指定回调的主机域名 (在反向代理下非常有用)
RP_ORIGIN=https://您的后台域名.com
```

### 回调 URL 设置

在 GitHub Developer Settings 或 Google Cloud Console 配置客户端凭证时，请务必注册以下回调 Redirect URI：

- **GitHub 回调**: `https://您的后台域名.com/api/auth/oauth/github/callback`
- **Google 回调**: `https://您的后台域名.com/api/auth/oauth/google/callback`

### 关闭注册时的安全行为

如果您在“系统设置”中关闭了新用户注册（即 `registrationOpen: false`）：
- **已绑定的账号**：已经绑定过的 Google 或 GitHub 账号仍可正常一键登录。
- **自动邮箱绑定**：如果该第三方账户的邮箱与您已有的本地管理员账户邮箱一致，系统会自动关联绑定该第三方账户以保护访问权限。
- **阻止新注册**：全新用户（未在系统中有任何关联邮箱或绑定的陌生人）在登录页尝试通过 Google/GitHub 登录时，系统将**直接予以拦截**，禁止自动创建账户，并重定向回登录页给出温和的错误提示。
