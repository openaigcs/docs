AIGCS supports advanced passwordless authentication and social logins using GitHub and Google.

## Passkeys (WebAuthn)

Passkeys allow you to securely sign in using biometric authentication (e.g., Apple Touch ID / Face ID, Windows Hello, Android Fingerprint) or password managers (e.g., Bitwarden, 1Password).

### Prerequisites

- Passkeys require a secure context. They will **only work** over `HTTPS` or on `localhost` during local development.
- No external environment configuration is required.

### Setup Instructions

1. Log in to your AIGCS dashboard using your email/username and password.
2. Navigate to **Profile Settings** (by clicking your avatar at the bottom-left corner).
3. Under the **Passkeys** card, click **Add Passkey**.
4. Follow your browser's prompt to verify your biometric identity or configure your password manager.
5. Once verified, the passkey will appear in your registered keys list. You can now use it on the login page by clicking **Login with Passkey**.

## Third-party Social Logins (OAuth 2.0)

AIGCS supports一键登录 (one-click shortcut login) through GitHub and Google.

### Environment Variables

Configure the following environment variables in your server settings or `.env` file to enable these login options:

```bash
# GitHub OAuth Configuration
GITHUB_CLIENT_ID=your_github_client_id
GITHUB_CLIENT_SECRET=your_github_client_secret

# Google OAuth Configuration
GOOGLE_CLIENT_ID=your_google_client_id
GOOGLE_CLIENT_SECRET=your_google_client_secret

# Optional: Override the redirect callback base URL (useful behind proxies)
RP_ORIGIN=https://your-admin-domain.com
```

### Callback Redirect URIs

When setting up your applications on GitHub Developer Settings or Google Cloud Console, register the following callback Redirect URIs:

- **GitHub Callback**: `https://your-admin-domain.com/api/auth/oauth/github/callback`
- **Google Callback**: `https://your-admin-domain.com/api/auth/oauth/google/callback`

### Registration Closed Safety Handling

If you close public registration (by setting `registrationOpen: false` in Admin System Settings):
- **Linked Accounts**: Previously bound third-party accounts will continue to sign in successfully.
- **Auto-binding**: If an unlinked GitHub/Google account attempts to sign in, and its primary email matches an existing local AIGCS profile, it will be automatically bound to protect administration access.
- **New Sign-ins Blocked**: New users trying to register via Google or GitHub will be **blocked** from account creation and redirected back with a friendly registration-closed notice.
