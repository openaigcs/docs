**原生评论插件** (Native) 启用存储在 AIGCS 自身数据库中的访客评论。

## 特性

- **回复线程** — 嵌套评论回复
- **编辑窗口** — 可配置的时间限制（默认：3 分钟）
- **管理员 PIN** — 评论审核密码
- **邮件通知** — 新评论和回复提醒
- **Gravatar 支持** — 带代理选项
- **屏蔽关键词** — 自动拒绝包含指定词语的评论
- **邮箱域名过滤** — 白名单/黑名单邮箱域名

## 配置

原生评论插件在管理面板中有 29 个可配置设置：

| 设置 | 类型 | 默认值 | 描述 |
|---------|------|---------|-------------|
| `formPosition` | string | `top` | 评论表单位置（top/bottom） |
| `aiPosition` | string | `above` | AI 与访客评论顺序 |
| `requiredFields` | string[] | `["author","email"]` | 必填表单字段 |
| `gravatarProxy` | string | — | Gravatar 代理 URL |
| `blockedKeywords` | string[] | `[]` | 自动拒绝关键词 |
| `notifyOnReply` | boolean | `true` | 邮件回复通知 |

## 启用

1. 前往管理面板的**插件**
2. 在插件列表中找到**原生评论插件**
3. 切换启用（更改立即生效）

## 站长身份防冒用保护 (Admin PIN)

原生插件强制提供防身份冒用功能。当访客试图在前端留言表单中填写属于“管理员列表中的邮箱”时，系统会要求他们必须提供正确的 **Admin PIN** 码（可在后台设置）。如果未能提供正确的 PIN 码，提交将被拒绝。这确保了其他人无法在评论区冒充站长发言。

## 删除评论机制与安全

AIGCS 为原生评论插件设计了严格的安全软删除机制，确保隐私安全与防滥用：

1. **所有权校验**：访客必须使用最初发表评论时的真实邮箱申请删除。
2. **邮箱验证码验证**：系统会向该邮箱下发 6 位数的高强度、一次性验证码，有效期 30 分钟。强依赖于后台正确配置了 SMTP 服务。
3. **软删除与隐私擦除**：为防止评论嵌套树断裂，验证成功后系统执行“软删除”。评论内容变为“此评论已被作者删除”，并在底层**彻底擦除该评论记录的作者邮箱和网址等个人隐私数据 (PII)**。
4. **防爆破机制**：验证码单次输错立刻作废。24 小时内单一邮箱最多申请 5 次，且每次发送强制冷却 120 秒，从根源上杜绝恶意轰炸与暴力破解。
