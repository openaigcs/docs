## 服务器问题

### 服务器无法启动

**检查：** 是否设置了 `JWT_SECRET`。
**解决方案：** 生成随机密钥并在 `.env` 或环境变量中设置。

### 数据库连接失败

**检查：** `DATABASE_URL` 格式以及数据库服务器是否运行。
**解决方案：**
- SQLite：确保 `/app/data/` 目录存在且可写

### 端口已被占用

**解决方案：** 更改 `PORT` 环境变量或停止冲突的进程。

### 权限被拒绝（Docker）

**解决方案：** 使用提供的 `volumes` 映射，不要更改内部用户。

## AI 提供商问题

### 提供商测试超时

**检查：** API 端点 URL 和网络连接。
**解决方案：**
- 验证 API 密钥是否正确
- 检查提供商端点是否可从您的服务器访问
- 对于 Ollama，确保它在本地运行

### "Model not found" 错误

**解决方案：** 更新模型名称。通过 `listModels` API 检查提供商当前的模型列表。

### 评论未生成

**检查：** 页面缓存状态。
**解决方案：**
- 确保页面 URL 可从服务器访问
- 检查内容选择器是否匹配页面结构
- 验证至少有一个提供商已启用并配置

## Widget 问题

### Widget 未显示

**检查：** 浏览器控制台错误信息。
**解决方案：**
- 验证 `domain` 是否与管理面板中的站点匹配
- 确保 `server` URL 正确且可访问
- 检查 CORS 设置是否允许您的域名

### Widget 显示 403

**解决方案：**
- 站点域名与 widget 的域名不匹配
- 检查 `ALLOWED_ORIGINS` 配置
- 如果使用代理，确保 Host 请求头已转发

### 主题未正确应用

**解决方案：** 确保主题名称有效。参见 [Widget 主题](../guide/widget#theme-system) 章节。

## SMTP 问题

### 测试邮件发送失败

**检查：** SMTP 设置和认证信息。
**解决方案：**
- 验证主机、端口和认证方法
- 对于 Gmail，使用应用专用密码
- 检查端口 587（STARTTLS）或 465（SSL）是否正确

### 邮件被标记为垃圾邮件

**解决方案：**
- 配置 SPF/DKIM/DMARC DNS 记录
- 使用信誉良好的 SMTP 提供商（SendGrid、Mailgun 等）

## CORS 问题

### 请求被 CORS 阻止

**解决方案：**
- 将您的域名添加到 `ALLOWED_ORIGINS`
- 检查浏览器控制台中被阻止的确切来源
- 开发环境可使用 `*`（生产环境不推荐）

## 请求头

### X-Requested-With 错误

**解决方案：** 所有管理 API 请求必须包含请求头 `X-Requested-With: XMLHttpRequest`。管理面板自动处理此问题。
